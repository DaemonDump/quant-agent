# 量化交易系统

一套完整的模块化量化交易系统，适配手动买卖操作，支持多标的实时监控和策略优化。

## 系统特点

- **模块化设计**：6大独立模块，职责清晰，便于调试和维护
- **手动操作适配**：跳过交易接口，完全适配手动买卖需求
- **实时数据接入**：基于Tushare接入A股实时行情数据
- **多因子+机器学习**：综合策略，提升信号准确性
- **可视化展示**：Web界面实时展示信号和K线图
- **完整监控体系**：实时监控、交易记录、应急处理、迭代优化

## 模块架构

### 模块1：策略顶层设计模块
- 负责机器学习模型训练和预测
- 支持逻辑回归、决策树等模型
- 输出涨跌概率，辅助信号生成

### 模块2：实时数据接入与处理模块
- Tushare数据采集
- 数据清洗和存储
- 数据校验和异常提醒

### 模块3：策略逻辑拆解模块
- 多因子计算（估值、趋势、资金）
- 信号生成和过滤
- 交易触发逻辑

### 模块4：信号输出模块
- Flask后端API
- Vue3前端界面
- ECharts图表展示
- WebSocket实时推送

### 模块5：回测与风险验证模块
- 回测框架
- 参数优化
- 风险测试
- 过拟合检查

### 模块6：实盘监控与运维迭代模块
- 实时监控
- 交易记录
- 应急处理
- 迭代优化

## 安装步骤

### 1. 环境要求
- Python 3.8+
- MySQL 5.7+
- Tushare账号

### 2. 安装依赖
```bash
pip install -r requirements.txt
```

### 3. 配置系统
1. 复制 `.env.example` 为 `.env`
2. 修改 `.env` 文件，配置以下参数：
   - TUSHARE_TOKEN: Tushare API Token
   - MYSQL_HOST: 数据库地址
   - MYSQL_USER: 数据库用户名
   - MYSQL_PASSWORD: 数据库密码
   - MYSQL_DATABASE: 数据库名称

### 4. 初始化数据库
```bash
python init_db.py
```

## 使用说明

### 启动系统
```bash
python main.py
```

启动菜单：
1. 完整启动（数据采集+监控+Web服务）
2. 仅启动Web服务
3. 仅启动数据采集和监控
4. 回测模式
5. 初始化数据库
0. 退出

### 访问Web界面
启动后访问：http://localhost:5000

## 策略配置

### 核心参数
- 年化收益目标：20%
- 最大回撤：10%
- 总仓位上限：80%
- 单票仓位上限：10%
- 每日交易限制：10次
- 每周交易限制：50次

### 信号阈值
- 买入评分阈值：8分
- 卖出评分阈值：3分
- 买入概率阈值：70%
- 卖出概率阈值：70%
- 止损阈值：5%
- 止盈阈值：10%

### 因子权重
- 估值因子：30%
- 趋势因子：40%
- 资金因子：30%

## 迭代优化

### 定期复盘
建议每周进行一次策略复盘：
1. 查看实盘绩效
2. 对比回测结果
3. 生成优化建议
4. 调整策略参数

### 参数优化
使用模块5的参数优化功能：
- 因子权重优化
- 信号阈值优化
- 模型参数优化

## 注意事项

1. **数据源限制**：Tushare免费版有请求频率限制
2. **交易时间**：仅在交易时间内生成信号
3. **手动操作**：所有交易需手动执行，系统仅提供信号
4. **风险控制**：严格遵守仓位和交易频率限制
5. **数据备份**：定期备份数据库

## 故障排查

### 数据采集失败
- 检查网络连接
- 确认Tushare Token是否有效
- 查看日志文件

### Web服务无法启动
- 检查端口5000是否被占用
- 确认依赖包是否安装完整

### 数据库连接失败
- 确认MySQL服务是否运行
- 检查数据库配置是否正确
- 验证用户权限
